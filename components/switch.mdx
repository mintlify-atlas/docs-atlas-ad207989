The `switch:` component represents any on/off output that can be commanded by the user, an automation, or a home automation platform like Home Assistant. GPIO relays, template flags, device restart controls, UART command sequences, and Modbus coils all use this component type. This page covers every base configuration key, restore behaviour options, automation triggers, available actions, and platform-specific examples.

## Base Configuration

```yaml
switch:
  - platform: <platform_name>
    name: "My Switch"             # Entity name exposed to HA / MQTT
    id: my_switch                 # C++ variable ID for automations
    icon: "mdi:toggle-switch"     # MDI icon string
    entity_category: ""           # "" | diagnostic | config
    device_class: switch          # "" | outlet | switch
    restore_mode: ALWAYS_OFF      # Boot/restore behaviour (see below)
    inverted: false               # Invert the output signal
    on_turn_on: ...               # Automation triggers
    on_turn_off: ...
    on_state: ...
```

### Restore Modes

`restore_mode` controls what state the switch enters at boot or after a power cycle.

| Value | Behaviour |
|---|---|
| `RESTORE_DEFAULT_OFF` | Restore last state from flash; default to OFF if no saved state |
| `RESTORE_DEFAULT_ON` | Restore last state from flash; default to ON if no saved state |
| `RESTORE_INVERTED_DEFAULT_OFF` | Restore inverted last state; default to OFF |
| `RESTORE_INVERTED_DEFAULT_ON` | Restore inverted last state; default to ON |
| `ALWAYS_OFF` | Always start OFF regardless of saved state (**default**) |
| `ALWAYS_ON` | Always start ON regardless of saved state |
| `DISABLED` | Do not restore or set any initial state |

<Note>
The platform default `restore_mode` is `ALWAYS_OFF`. GPIO switches commonly use `RESTORE_DEFAULT_OFF` to remember their last state across reboots.
</Note>

### Device Classes

The switch component supports three device classes sourced from `const.py`:

| Value | Home Assistant icon |
|---|---|
| `""` (empty) | Generic switch |
| `outlet` | Power outlet |
| `switch` | Toggle switch |

### `inverted`

Setting `inverted: true` flips the underlying output signal. The switch state visible in Home Assistant remains logical (ON/OFF), but the GPIO or other output is physically inverted. This is useful for active-low relays.

## Automation Triggers

<CardGroup cols={3}>
  <Card title="on_turn_on" icon="toggle-on">
    Fires when the switch transitions to the ON state. No variables.
  </Card>
  <Card title="on_turn_off" icon="toggle-off">
    Fires when the switch transitions to the OFF state. No variables.
  </Card>
  <Card title="on_state" icon="circle-dot">
    Fires on any state change. Variable `x` (bool) holds the new state.
  </Card>
</CardGroup>

```yaml
switch:
  - platform: gpio
    pin: GPIO5
    name: "Relay"
    on_turn_on:
      then:
        - logger.log: "Relay energised"
    on_turn_off:
      then:
        - logger.log: "Relay de-energised"
    on_state:
      then:
        - logger.log:
            format: "Relay state: %s"
            args: ['x ? "ON" : "OFF"']
```

## Actions

Three actions are available to control any switch from automations or scripts.

<Steps>
  <Step title="switch.turn_on">
    Turn the switch on.

    ```yaml
    action:
      - switch.turn_on: my_switch
    ```
  </Step>
  <Step title="switch.turn_off">
    Turn the switch off.

    ```yaml
    action:
      - switch.turn_off: my_switch
    ```
  </Step>
  <Step title="switch.toggle">
    Toggle the switch (on → off, off → on).

    ```yaml
    action:
      - switch.toggle: my_switch
    ```
  </Step>
</Steps>

A fourth `switch.control` action accepts a templatable boolean for programmatic control:

```yaml
action:
  - switch.control:
      id: my_switch
      state: !lambda "return id(some_sensor).state > 30.0;"
```

## Platform Examples

<Tabs>
  <Tab title="GPIO Relay">

The most common switch platform — drives a GPIO pin that controls a relay module.

```yaml
switch:
  - platform: gpio
    name: "Water Pump"
    pin: GPIO26
    restore_mode: RESTORE_DEFAULT_OFF
    inverted: false
    on_turn_on:
      then:
        - delay: 30min
        - switch.turn_off: water_pump   # safety auto-off
```

  </Tab>
  <Tab title="Template Switch">

A virtual switch backed by a lambda that you control through automations.

```yaml
switch:
  - platform: template
    name: "Night Mode"
    id: night_mode
    restore_mode: RESTORE_DEFAULT_OFF
    optimistic: true              # report state immediately without confirming
    turn_on_action:
      - light.turn_on:
              id: hall_light
              brightness: 10%
    turn_off_action:
      - light.turn_off: hall_light
```

  </Tab>
  <Tab title="Restart / Safe Mode">

Special system switches that trigger device restarts.

```yaml
switch:
  - platform: restart
    name: "Restart Device"
    entity_category: config

  - platform: safe_mode
    name: "Safe Mode Boot"
    entity_category: config
```

  </Tab>
  <Tab title="UART Command Switch">

Send different byte sequences over UART when the switch turns on or off.

```yaml
switch:
  - platform: uart
    name: "LED Panel"
    uart_id: uart_bus
    data_on: [0xA0, 0x01, 0x01, 0xA2]
    data_off: [0xA0, 0x01, 0x00, 0xA1]
    restore_mode: ALWAYS_OFF
```

  </Tab>
  <Tab title="Modbus Controller">

Expose a Modbus coil as an ESPHome switch.

```yaml
switch:
  - platform: modbus_controller
    modbus_controller_id: modbus_hub
    name: "Pump Enable"
    address: 0x00
    register_type: coil
    restore_mode: RESTORE_DEFAULT_OFF
```

  </Tab>
</Tabs>

## Common Patterns

<AccordionGroup>

<Accordion title="Timed output with auto-off">

Turn a switch on for a fixed duration then automatically turn it off.

```yaml
switch:
  - platform: gpio
    pin: GPIO4
    name: "Sprinkler Zone 1"
    id: sprinkler_1
    restore_mode: ALWAYS_OFF

automation:
  - trigger:
      platform: time
      at: "06:00:00"
    action:
      - switch.turn_on: sprinkler_1
      - delay: 5min
      - switch.turn_off: sprinkler_1
```

</Accordion>

<Accordion title="Interlocked switches">

Ensure that turning one switch on turns another off (mutual exclusion).

```yaml
switch:
  - platform: gpio
    pin: GPIO12
    name: "Valve A"
    id: valve_a
    on_turn_on:
      then:
        - switch.turn_off: valve_b

  - platform: gpio
    pin: GPIO13
    name: "Valve B"
    id: valve_b
    on_turn_on:
      then:
        - switch.turn_off: valve_a
```

</Accordion>

</AccordionGroup>
